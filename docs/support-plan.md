# Support & Maintenance Plan

**Project:** GWA Calculator & Academic Tracker
**Documentation Lead:** Seth Palgan & Lorenz Lacanaria

## 1. Issue Reporting Process
Users and beta testers encountering bugs or requesting features must use the **GitHub Issues** tracker in our repository. 
* Users must provide: Expected behavior, actual behavior, and steps to reproduce the error.
* All issues will be tagged as `bug`, `enhancement`, or `documentation`.

## 2. Response Time Service Level Agreements (SLAs)
We prioritize issues based on severity:

| Severity Level | Definition | Target Response Time | Target Resolution Time |
| :--- | :--- | :--- | :--- |
| **High (Critical)** | App crashes, incorrect GWA calculation, data loss. | Within 12 Hours | 24 - 48 Hours |
| **Medium** | Minor UI bugs, typos, feature requests. | Within 24 Hours | Next Sprint Cycle |
| **Low** | Dark mode color adjustments, text changes. | Within 3 Days | Backlogged |

## 3. Escalation
If a critical database or backend issue occurs, the issue is immediately escalated to **James Henry Emorricha** (Database Administrator) and **Nash Andrew Bondoc** (Lead Developer).
